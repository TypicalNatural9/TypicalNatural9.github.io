2021-04-08

Went through the first lecture.

Bitcoin uses Unspent Transaction Output (UTxO) model. Address can use UTxO's when transferring money. The entire UTxO has to be spent. If it is smaller, then the address sends the remaining money back to itself.

Cardano uses extended UTxO where addresses can contain scripts instead. Ethereum's model is simpler for humans to understand because it tells directly how much someone has money and arbitrary amount can be spent. UTxO is better for concurrent/distributed systems. Lars also went through how Ethereum's account model and smart contracts can see entire state of the blockchain which can have side effects.

Lars showed the Plutus playground and did an auction demo. Playground worked well with the demo. I know the basics of Haskell, but the Plutus script looked slightly complicated. Homework was just to install playground and play with the script.
